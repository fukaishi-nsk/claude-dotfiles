## プロジェクト伴走の開始

現在のフォルダを新しい案件プロジェクトとして認識し、以下の手順を実行せよ。

### Step 1: ナレッジベースの確認
- `02_Plan/` にナレッジベース（`プロジェクト概要_*.md`）が既にあるか確認する
- **ある場合** → 読み込んで内容を把握し、最終更新日以降の変更がないか確認する
- **ない場合** → Step 2 へ

### Step 2: 資料の読み込み
以下の優先順で全資料を読み込む：
1. `06_Recording/` — 会議の文字起こし（時系列順）
2. `00_File_from/` — もらった資料
3. `09_契約/` — 契約関連
4. その他フォルダに資料があれば読む

### Step 3: ナレッジベース作成
読み込んだ内容を `02_Plan/プロジェクト概要_[案件名].md` にまとめる。構成：

```
# プロジェクト概要: [案件名]

## 基本情報
- クライアント:
- 案件概要:
- スケジュール:
- 予算感:

## 決定事項
（会議等で確定したこと）

## 未決事項
（まだ決まっていないこと・確認が必要なこと）

## 関係者
（誰が何の担当か）

## 更新ログ
- [日付] 初回作成
```

### Step 4: 不明点の質問
- 資料から読み取れなかった情報があればユーザーに質問する
- 推測で埋めない。わからないことは「わからない」と明記する
